That list covers the primary ways FastAPI receives and processes data from a client request, categorized by where the data is located:

1. Single Input element or a form data: Data sent via an HTML form submission (e.g., a text field, checkbox, or file).

--FastAPI uses: Form(...) and File(...) dependencies, often with the request body having the content type multipart/form-data or application/x-www-form-urlencoded.

2. JSON format: A structured data object sent in the request body.

--FastAPI uses: A Pydantic model to define and validate the expected structure. This is the default for POST, PUT, and PATCH requests. The content type is typically application/json.

3. Query parameters: Key-value pairs appended to the URL after a question mark (?). Used mainly for filtering, pagination, or optional parameters.

--FastAPI uses: Function parameters that are not defined as path parameters and are not part of the body.

4. Path parameters: Variables defined directly within the URL path, used to identify a specific resource.

--FastAPI uses: Variables declared in the path decorator, e.g., @app.get("/items/{item_id}").
